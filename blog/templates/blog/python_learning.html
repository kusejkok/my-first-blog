Working through the book "Automate the boring stuff with Python"

Chapter 1: Python Basics

  • Variable names in Python can contain letters, numbers and _, but must start with a letter
  • Variables are assigned with =
  • Python uses the common operators: +, -, *, /. // gives back the integer part of the division, % the remainder. ** powers.
  • str(), int(), float() will evaluate the input as string, integer and float.
  • print() displays the argument.
  • len() gives the length of a string.
  • input() asks the user for input.

Chapter 2: Flow control

  • Boolean values in Python are "True" and "False".
  • The three Boolean operators are "and", "or" and "not".
  • The six comparison operators are: ==, !=, <, <=, >, >=
  • A Python program can be exited with ctr C; or sys.exit() in the code.
  • The function range() can take 1, 2 or 3 arguments. (from, to by)
  • Importing packages in Python needs "import ..." at the beginning, e.g., import random
  • Function from an imprted module are called with module.function(), e.g., random.randint(5)
  • if condition: do..., elif, else
  • while condition: do..., you can break with "break" or continue with "continue"
  • for i in range(5): condition...
  • input() asks the user for input.

Chapter 3: Functions

  • A function is defined with def functionName():
  • Return values are given with the return command.
  • The None value: E.g. print() has no return value, it returns None.
  • Python distinguishes between local and global values: local values within functions are deleted after the function call.
  • Within functions, Python does not fall back to globally defined variables, e.g.
    eggs = "hi"
    def spam():
    print(eggs)
    eggs = "hallo"
    spam()
    would give an error message: UnboundLocalError
  • The try: except clause can avoid function being stopped because of errors.

Chapter 4: Lists

  • Lists are created with brackets: [...]
  • Lists indexing starts at 0. list[0:i] does not include the elemt indexed with i.
  • list[:i] gives the list elements 0 to i-1, list[i:] gives list elements i until the last one.
  • Lists of lists are indexed with list[i][j].
  • Negative indexing starts at -1, i.e., list[-1] gives the last element.
  • Lists can be concatenated with + and replicated with *
  • del list[i] removes the list element with index i.
  • in and not in can be used to check whether elements are part of a list.
  • Lists can be used to simultaneously assign values to many variables, like a,b,c = ["hi", "hallo", "goodby"]
  • +=, *=, -=, /=, %= are shortcuts for list = list + ..,...
  • Methods are functions specific for a data type, and called with list.method()
  • Methods for lists are: index(element), append(elemet), insert(index, element), remove(element)
  • Careful with remove and index: only gives back information on the first appearance
  • The sort() method uses ASCIIbetical order, i.e., uppercase A-Z, then lower case a-z; sort(key = str.lower) helps for alphabetical sorting
  • Applying methods on lists change the immediately, myList = myList.append("hallo") gives back None.
  • Tuples and strings are list-like but immutable
  • To create a tuple with one elements, one needs a trailing comma, i.e., myTuple = ("Hallo",)
  • str(), tuple() and list() can be used to change the type
  • Variables store reference to lists, not the list itself!
  • copy() and deepcopy() can be used to make a copy of a list (or a list of lists), not only reference the list.

Chapter 5: Dictionaries and structuring data

  • While lists are ordered, dictionaries are unordered.
  • dict = {key1: value1, key2: value2,..., keyN, valueN}
  • Values in dictionaries can be accessed by dict[key1].
  • The methods keys(), values() and items() access the keys, values and items (tuples of keys and values)
  • 'eggs' in picnicItems is the same as 'eggs' in picnicItems.keys()
  • The method get() either gets the values of the given key or returns a default value: picnicItems.get('eggs', 0)
  • The setdefault() method can set a default value, but not changing the value in case the key exists.
  • Dictionaries can contain dictionaries as values

Chapter 6: Manipulating strings

  • Escape characters in strings are: \', \", \t, \n, \\.
  • print(r"blabla\nblabla") will print escape characters.
  • Multiline strings """ consider the formatting (tabs, new lines) when printing.
  • The in and not in operators work for strings too! ("Hallo" in "Hallo du")
  • The string methods .upper(), .lower() convert to upper/lower-case string only.
  • The isX methods (.isupper(), .islower(), isalpha(), isalnum(), isdecomal()) return a Boolean value.
  • The methods .startswith() and .endswith() check for string beginnings and endings.
  • Splitting strings: "My name is Kathi.".split("a") gives ["My n", "me is K", "thi."]
  • Joining strings: "a".join(["My n", "me is K", "thi."]) gives "My name is Kathi."
  • .rjust(n, "*"), ljust(n, "*") and center(n, "*") insert "*" to the left, right or both sides, to make the string n characters long.

Errors:

  • ZeroDivisionError
  • NameError (e.g. you refer to a variable name that does not exist)
  • TypeError (e.g. using a list method on a string)
  • UnboundLocalError (e.g. defining a local variable only after it is called in a function)
  • ValueError (e.g. using the method index calling an index that does not exist in a list)
  • AttributeError (e.g. using the method append on a string, not list)
  • KeyError (e.g. Asking for a key in a dictionary that is not defined)
